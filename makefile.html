Makefile dokumentace

Teorie: Linkování a typová bezpečnost

Při přidání C++ už Makefile není jen o kompilaci, ale o správě vazeb mezi symboly. C++ kompilátor mění názvy funkcí (Name Mangling), aby podpořil overloading. Pokud voláte funkci z ASM nebo C, musíte použít extern "C", jinak linker (LD) nahlásí undefined reference. Kolikrát u kompilace více než jednoho jazyka se používá LD := [ten co má nejvíce v sobě]. Pro C a ASM se použije gcc a pro C++ a ASM se použije g++.

Makefile funguje na základní struktuře a tudíž první se nacházý delarace.


Krok 1 - Deklarace nástrojů:

Pro projekty obsahující C++ je **nutné** použít g++ jako finální linker, protože automaticky přibaluje C++ runtime.

  • CC := gcc - Kompilátor pro C.
  • CXX := g++ - Kompilátor pro C++.
  • AS := nasm - Assembler.

Krok 2 - Nastavení flagů (Specifika C++):

Nastavení flagů se dává podstatě jako kdyby byli konpilovány soubory ručně, vždy musí být přidány tzv. flagy které určují způsob kompilace.

  • CFLAGS := -Wall -Wextra -g - Standardní flagy pro C.
  • CXXFLAGS := -Wall -Wextra -std=c++20 -g - -std=c++20 definuje normu jazyka, což je zásadní pro konzistenci syntaxe.
  • ASFLAGS := -f elf64 -g - Ladicí informace i pro assembler.

Krok 3 - Správa souborů a cílů:

Využíváme substituci, abychom automaticky generovali seznam všech objektových souborů z různých přípon.

  • TARGET := program
  • SRCS_C := $(wildcard *.c)
  • SRCS_CXX := $(wildcard *.cpp)

  • SRCS_ASM := $(wildcard *.s)
  • OBJS := $(SRCS_C:.c=.o) $(SRCS_CXX:.cpp=.o) $(SRCS_ASM:.s=.o)

Krok 4 - Linkování:

Linkování (Linking) je závěrečná fáze, kde linker vezme izolované objektové soubory (.o) a vyřeší jejich vzájemné reference. Tento proces transformuje kód z "rozsypaného stavu" do finálního, spustitelného nebo znovupoužitelného produktu. Aby byl uživatel schopen vytvořit Makefile pro cokoliv, musí znát tyto čtyři fundamentální způsoby spojování.

 Makefile

$(TARGET): $(OBJS)
    $(CC) $(CFLAGS) -o $(TARGET) $(OBJS)

Prvek Význam v tomto zápisu Ekvivalent (Metoda A)
$(TARGET) Název binárky (např. program). $@
$(OBJS) Seznam všech .o souborů. $^
$(CC) Kompilátor použitý i jako linker. $(LD)
$(CFLAGS) Předání parametrů (debug, optimalizace). $(LDFLAGS)

Způsob A: Standardní binární program (Executable)

Nejčastější způsob pro tvorbu aplikací. Linker spojí objekty a přidá k nim kód pro spuštění operačním systémem (vstupní bod _start nebo main).

 Makefile

$(TARGET): $(OBJS)
    $(LD) $(LDFLAGS) -o $@ $^ $(LIBS)

Kritické proměnné:

  • $@ (Target): Název výsledného programu (vlevo od dvojtečky).
  • $^ (Dependencies): Seznam všech objektů (vpravo od dvojtečky). Pokud použijete $<, linker uvidí jen první soubor a nahlásí chybu undefined reference pro zbytek.

Způsob B: Polyglotní/Hybridní linkování (C + C++ + ASM)

Univerzální metoda pro projekty míchající různé jazyky. Zde je volba linkeru kritická.

 Makefile

LD := g++ # Pro C++ standardní knihovny
$(TARGET): $(OBJS_C) $(OBJS_CPP) $(OBJS_ASM)
    $(LD) $(LDFLAGS) -o $@ $^

Chyták: I když máte většinu kódu v C, pokud obsahuje byť jeden C++ modul, musíte linkovat přes g++ (nebo clang++). gcc neumí samo přibalit libstdc++, což vede k selhání linkování.

Způsob C: Statická knihovna (Static Library - .a)

Pokud netvoříte program, ale balíček pro jiné vývojáře. V tomto případě linker ld nahrazujeme nástrojem ar (archiver).

 Makefile

$(LIB_NAME).a: $(OBJS)
    ar rcs $@ $^

Parametry rcs: r (replace) nahradí staré objekty, c (create) vytvoří archiv, s (index) vytvoří symbolický index pro rychlejší linkování v budoucnu.

Způsob D: Sdílená/Dynamická knihovna (Shared Object - .so)

Pro knihovny, které se do paměti načítají až při spuštění programu. Vyžaduje speciální instrukce pro linker.

 Makefile

$(LIB_NAME).so: $(OBJS)
    $(LD) -shared -fPIC -o $@ $^

Pozor: Všechny objekty v $(OBJS) musí být v Kroku 5 zkompilovány s flagem -fPIC (Position Independent Code), jinak linkování v Kroku 4 selže.

Způsob E: Kombinovaný přístup (Hybrid)

Pro maximální univerzálnost a čitelnost se často dokumentuje i tento způsob, který kombinuje explicitní cíl s automatickými vstupy:

 Makefile

$(TARGET): (OBJS)
(CC) -o $@ $^ (CFLAGS) $(LDFLAGS) $@\
# nahrazuje \(TARGET), čímž se eliminuje riziko překlepu při přejmenování binárky.


Krok 5 - Vzorová pravidla pro všechny jazyky:

Tento krok definuje, jak se ze zdrojového textu (C, C++, ASM) stane binární polotovar – objektový soubor (.o). Existuje pět hlavních způsobů, jak toto v Makefile zapsat. Každý má své specifické využití, výhody a kritické chytáky.

Způsob A: Explicitní pravidla (Ruční výpis)

Nejzákladnější a nejstarší metoda. Každý soubor má svůj vlastní řádek. Poskytuje absolutní kontrolu, ale je neudržitelná pro velké projekty.

 Makefile

main.o: main.c
    $(CC) $(CFLAGS) -c main.c -o main.o
utils.o: utils.c
    $(CC) $(CFLAGS) -DDEBUG -c utils.c -o utils.o

Výhoda: Umožňuje přidat specifické flagy (např. -DDEBUG) pouze pro jeden konkrétní soubor.

Způsob B: Pattern Rules (Moderní GNU Make)

Využívá zástupný znak %. Je to šablona: "Pokud vidíš cíl končící na .o a máš k němu .c, použij tento postup."

 Makefile

%.o: %.c
    $(CC) $(CFLAGS) -c $< -o $@

Hloubková definice proměnných:

  • $@: Název cíle (target), např. main.o.
  • $<: Název první prerekvizity (zdroj), např. main.c.

Způsob C: Suffix Rules (Historický / POSIX)

Zastaralý způsob, se kterým se setkáte ve starých systémech. Nepoužívá %, ale definuje transformaci přípon. Před tímto pravidlem se musí definovat seznam povolených přípon .SUFFIXES.

 Makefile

.SUFFIXES: .c .o
.c.o:
    $(CC) $(CFLAGS) -c $< -o $@

Chyták: Suffixová pravidla neumí pracovat s více prerekvizitami (např. závislost na hlavičkovém souboru) tak elegantně jako pattern rules.

Způsob D: Static Pattern Rules (Omezený výběr)

Kombinace explicitního výpisu a šablony. Říkáte: "Z tohoto konkrétního seznamu souborů udělej objekty pomocí této masky."

 Makefile

$(OBJS_C): %.o: %.c
    $(CC) $(CFLAGS) -c $< -o $@

Proč to chtít? Pokud máte v jedné složce soubory pro Windows i Linux, touto metodou zajistíte, že se budou kompilovat jen ty, které jste vypsali v $(OBJS_C), a ne všechno, co končí na .c.

Způsob E: Implicitní pravidla (Vestavěná)

Make má v sobě zabudovanou databázi pravidel. Pokud v Makefile nenapíšete žádný Krok 5, Make se pokusí soubor zkompilovat sám pomocí předdefinovaných maker $(CC) a $(CFLAGS).

 Makefile

# V Makefile není nic, jen:
main.o: main.c
# Make automaticky spustí: cc -c -o main.o main.c

Krok 6 - Úklid:

 Makefile

.PHONY: all clean
clean:
  rm -rf $(OBJS) $(TARGET) *.d
